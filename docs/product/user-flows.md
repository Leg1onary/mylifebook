# User Flows — MyLifeBook

## Принципы

- Каждый flow описывает **реальный сценарий использования**, а не просто последовательность экранов.
- Все flows написаны от первого лица — так проще держать в голове при реализации.
- Каждый flow имеет **happy path** и список edge cases.
- Flows охватывают только MVP-модули.

---

## Flow 1. Первый вход и онбординг

### Контекст
Пользователь открывает mylifebook.ru впервые после деплоя.

### Happy path

```
1. Открываю mylifebook.ru
2. Вижу LoginPage — форма логин + пароль
3. Ввожу credentials (заранее заданные в .env / seed)
4. Попадаю на TodayPage
5. Вижу онбординг-баннер: «Привет. Добавь личный контекст, чтобы AI лучше тебя понимал»
   → [Добавить сейчас] или [Позже]
6. Если [Позже] → баннер скрывается, остаюсь на TodayPage
7. Если [Добавить сейчас] → /profile/context
8. На PersonalContextPage вставляю raw text о себе из внешнего чата
9. Нажимаю [Извлечь и обновить профиль]
10. AI (через OpenRouter) извлекает структурированный профиль
11. Вижу extracted профиль — редактирую если нужно → [Сохранить]
12. Возвращаюсь на TodayPage — готов к работе
```

### Edge cases
- Неверный пароль → inline error под полем, не очищает логин
- Пустой пароль → валидация до отправки
- Ошибка AI при извлечении профиля → показываю ошибку, предлагаю ввести профиль вручную
- Пользователь пропустил онбординг → в Settings всегда доступен /profile/context

## Flow 2. Утренний check-in

### Контекст
Каждое утро. Занимает 1–2 минуты. Это основа регулярности.

### Happy path

```
1. Открываю приложение (с домашнего экрана — PWA)
2. TodayPage показывает «Сегодня ещё не отмечен» + CTA [Отметить день]
3. Нажимаю [Отметить день] → DailyCheckinPage
4. Выставляю шкалы 1–10: mood, energy, anxiety, shame, loneliness, anger
5. Выбираю emotion tags из предложенных
6. Заполняю (или пропускаю) текстовые поля:
   — «Что сегодня важно?»
   — «Что сейчас болит?»
   — «Что поддерживает?»
   — «Что мне нужно сегодня?»
7. Отвечаю на toggle: «Был триггер вчера вечером?» → Нет
8. Нажимаю [Сохранить]
9. Возвращаюсь на TodayPage
10. Карточка состояния обновилась: статус «день отмечен», сводка шкал
```

### Edge cases
- Нет интернета → черновик сохраняется локально, sync при восстановлении
- Чекин за сегодня уже есть → TodayPage показывает карточку с кнопкой [Изменить]
- Пользователь закрыл форму на полпути → спрашиваем «Сохранить черновик?»

## Flow 3. SOS — накрыло прямо сейчас

### Контекст
Острый момент. Пользователя накрыло, нужно зафиксировать быстро.
Этот flow должен работать за 60–90 секунд максимум.

### Happy path

```
1. Любой экран → нажимаю центральную + кнопку в bottom nav
2. Открывается modal-sheet с выбором действий
3. Выбираю [SOS — накрыло прямо сейчас]
4. Открывается SOS overlay (поверх текущего экрана)

   Шаг 1: «Что случилось?» + «Что я сейчас думаю?» — два коротких поля
   Шаг 2: Эмоции — быстрый мультивыбор тегов + один общий слайдер интенсивности 0–10
   Шаг 3: «Какой старый закон включился?» + стоп-фраза из профиля
   Шаг 4: Один grounding step (из заготовленных под профиль) + кнопка [Я зафиксировал]

5. Нажимаю [Я зафиксировал] → запись сохраняется как TriggerEvent (draft)
6. Показывается brief confirmation: «Зафиксировано. Вернись позже и разбери это подробно»
   → [Разобрать сейчас] | [Закрыть]
7. Если [Закрыть] → возвращаюсь туда, откуда открыл
8. Если [Разобрать сейчас] → ThoughtRecordWizardPage с pre-filled данными из SOS
```

### Edge cases
- Нет интернета → сохраняется в draftStore, sync позже
- Пользователь свайпает вниз не завершив → спрашиваем «Сохранить незаконченную запись?»
- На шаге 3 нет профиля → поле свободное без suggestions

## Flow 4. Полный цикл: триггер → thought record → эксперимент

### Контекст
Это центральный и самый ценный flow. Пользователь переживает что-то болезненное,
хочет разобраться и выбрать новое действие.

### 4а. Захват триггера

```
1. Что-то произошло. Открываю приложение.
2. Нажимаю + → [Записать триггер] → TriggerCapturePage
3. Заполняю:
   — «Что произошло?»
   — «Первая мысль»
   — Эмоции + интенсивность
   — «Реакция в теле» (опционально)
   — «Что хочется сделать прямо сейчас?»
   — «Какой старый закон включился?» (suggestions из профиля)
4. Нажимаю [Сохранить и разобрать] → ThoughtRecordWizardPage
   (данные из триггера pre-fill шаги 1, 2, 4, 6)
```

### 4б. Thought Record Wizard

```
5. ThoughtRecordWizardPage открывается с прогресс-баром «Шаг 1 из 12»
6. Шаги 1–5: часть уже заполнена из триггера, дополняю/уточняю
7. Шаг 6 — Старый закон: выбираю из suggestions или пишу своё
8. Шаги 7–8 — Доказательства за и против: думаю, пишу
9. Шаг 9 — Искажения: выбираю из списка что узнаю в этой ситуации
10. Шаг 10 — Новый взгляд: формулирую честную альтернативу
11. Шаг 11 — Новый шаг: что конкретно сделаю иначе
12. Шаг 12 — AI Reframe: нажимаю [Запросить у AI]
    → AI (через OpenRouter) анализирует всё + личный профиль
    → Возвращает персонализированный reframe
    → Читаю, могу принять или проигнорировать
13. Нажимаю [Завершить] → ThoughtRecordDetailPage
```

### 4в. Создание эксперимента

```
14. На ThoughtRecordDetailPage вижу кнопку [Создать эксперимент]
15. Нажимаю → ExperimentForm (pre-filled: старый закон из TR, новый шаг из TR)
16. Дополняю:
    — «Чего я конкретно боюсь?»
    — «Что попробую сделать иначе?» (уже есть из TR)
    — «Прогноз: что случится?»
    — Страх до (0–10)
    — Запланировать дату (опционально)
17. Сохраняю → ExperimentDetailPage
```

### 4г. Follow-up эксперимента

```
18. Через N дней: пуш-напоминание или просто открываю /experiments
19. Нахожу эксперимент → ExperimentDetailPage
20. Заполняю follow-up:
    — «Что реально произошло?»
    — «Что сбылось из прогноза?»
    — «Что не сбылось?»
    — «Чему это меня учит?»
    — Страх после (0–10)
21. Нажимаю [Завершить эксперимент]
22. Вижу сравнение: страх до vs страх после + что не сбылось
```

### Edge cases
- Пользователь прервал wizard → черновик автосохраняется каждый шаг
- AI недоступен на шаге 12 → показываю «AI недоступен, пропустить шаг?»
- AI вернул ошибку → показываю текст ошибки + кнопка «Попробовать снова»
- Эксперимент создан без TR → форма без pre-fill, всё вводится вручную

## Flow 5. Вечерняя рефлексия

### Контекст
Необязательно, но ценно. Пользователь возвращается к дню вечером.

### Happy path

```
1. Вечернее напоминание → открываю приложение
2. TodayPage — вижу карточку дня (чекин уже был утром)
3. Если был триггер-черновик из SOS → TodayPage показывает: «У тебя есть незавершённая запись»
   → [Разобрать] | [Удалить]
4. Если нет черновиков → могу открыть Daily Checkin и добавить вечерние поля
   (обновляю: «Что болит?», «Что поддержало?»)
5. Если был сложный день → иду в + → [Разобрать мысль] → новый TR
6. Закрываю приложение
```

## Flow 6. Weekly Review

### Контекст
Раз в неделю, обычно воскресенье вечером или понедельник утром.
Самый длинный и ценный ритуал.

### Happy path

```
1. Напоминание (или сам открываю) → [Week] таб → WeeklyReviewPage
2. По умолчанию открывается текущая (или последняя) неделя
3. Вижу автоматически собранные метрики:
   — Количество чекинов, триггеров, TR, экспериментов
   — Среднее настроение / тревога / стыд / одиночество
   — Топ-3 категории триггеров
   — Топ-3 старых закона
4. Нажимаю [Получить AI summary]
   → AI анализирует все записи недели + личный профиль
   → Возвращает: summary недели, доминирующие паттерны, знаки прогресса, вопрос на следующую неделю
5. Читаю AI summary
6. Прохожу guided questions (6 вопросов, textarea):
   — «Где неделя ударила сильнее всего?»
   — «Какой старый закон включался чаще всего?»
   — «Где получилось не подчиниться ему?»
   — «Что оказалось не таким страшным?»
   — «Что я хочу проверить на следующей неделе?»
   — «Мой итог этой недели одной фразой»
7. Заполняю «Фокус следующей недели» (одно поле)
8. Нажимаю [Сохранить обзор]
9. TodayPage следующей недели покажет snippet этого фокуса
```

### Edge cases
- Меньше 3 чекинов за неделю → показываю «Мало данных, но можно заполнить вручную»
- AI недоступен → guided questions и текстовые поля всё равно доступны
- Пользователь начал и не закончил → draft автосохраняется
- Обзор за прошлую неделю → навигация ← → между неделями, прошлые обзоры read-only

## Flow 7. Просмотр insights и паттернов

### Контекст
Пользователь хочет увидеть, как меняется динамика.
Обычно происходит после weekly review или когда хочется «посмотреть как дела».

### Happy path

```
1. [Week] таб → WeeklyReviewPage → link [Смотреть аналитику] → InsightsPage
   или напрямую через навигацию внутри [Week] таба
2. Вижу переключатель периода: 7д / 30д / 90д
3. Mood trend: линейный график по дням, вижу волны
4. Anxiety / Shame / Loneliness trend: три линии
5. Trigger category histogram: в каких контекстах чаще
6. Old laws frequency: какой закон включался чаще всего
7. Experiments bar: завершено / активно
8. Streak / regularity: цепочка дней с чекинами
9. Могу тапнуть на бар в histogram → фильтрованный список триггеров этой категории
10. Могу тапнуть на строку old laws → список связанных TR
```

### Edge cases
- Нет данных за период → «Недостаточно данных, добавь больше записей»
- Только один день данных → показываем только то, что можно, без графиков

## Flow 8. Загрузка и обновление личного контекста

### Контекст
Пользователь хочет дать AI более глубокое понимание себя,
вставив summary из внешнего чата.

### Happy path

```
1. Иду в [Me] таб → Settings → «Личный контекст» → /profile/context
   или из онбординг-баннера
2. Вижу текущий extracted профиль (если есть) или пустой экран с CTA
3. Перехожу в секцию «Загрузить контекст»
4. Вставляю raw text в textarea (например: ответ чата о моих паттернах)
5. Нажимаю [Извлечь профиль]
6. Спиннер → AI извлекает структурированный профиль из текста
7. Вижу результат:
   — Ключевые триггеры
   — Старые законы
   — Типичные искажения
   — Предпочтения в общении
   — Цели роста
8. Могу отредактировать любое поле вручную
9. Нажимаю [Сохранить профиль]
10. AI в следующих reframe и weekly summary будет использовать этот профиль

Шаг 11 (опционально): [Обновить профиль] — вставить новый текст и объединить с текущим
```

### Edge cases
- AI недоступен → предлагаем ввести профиль вручную (все поля открытые textarea)
- Текст слишком длинный → предупреждение, предлагаем сократить
- Пользователь нажал «Обновить» → спрашиваем «Заменить или объединить с текущим?»

## Flow 9. Export для терапевта

### Контекст
Пользователь идёт к психологу и хочет принести структурированный материал.

### Happy path

```
1. Settings → секция «Данные» → [Экспорт]
2. Выбираю формат: PDF / Markdown / JSON
3. Выбираю период: эта неделя / прошлая неделя / последние 30 дней / произвольный
4. Нажимаю [Скачать]
5. Файл содержит:
   — Топ-5 триггеров периода (краткое описание)
   — Все thought records (full или summary)
   — Эксперименты с результатами
   — Weekly review summary
   — Повторяющиеся паттерны
   — AI summaries (если запрашивались)
```

### Edge cases
- Нет данных за период → «За выбранный период нет записей»
- Ошибка генерации PDF → предлагаем Markdown как fallback
